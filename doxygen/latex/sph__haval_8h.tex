\subsection{Detailed Description}
H\+A\+V\+AL interface.

H\+A\+V\+AL is actually a family of 15 hash functions, depending on whether the internal computation uses 3, 4 or 5 passes, and on the output length, which is 128, 160, 192, 224 or 256 bits. This implementation provides interface functions for all 15, which internally map to three cores (depending on the number of passes). Note that output lengths other than 256 bits are not obtained by a simple truncation of a longer result; the requested length is encoded within the padding data.

H\+A\+V\+AL was published in\+: Yuliang Zheng, Josef Pieprzyk and Jennifer Seberry\+: \char`\"{}\+H\+A\+V\+A\+L -\/-\/ a one-\/way hashing algorithm with variable length
of output\char`\"{}, Advances in Cryptology -- A\+U\+S\+C\+R\+Y\+PT\textquotesingle{}92, Lecture Notes in Computer Science, Vol.\+718, pp.\+83-\/104, Springer-\/\+Verlag, 1993.

This paper, and a reference implementation, are available on the Calyptix web site\+: \href{http://labs.calyptix.com/haval.php}{\texttt{ http\+://labs.\+calyptix.\+com/haval.\+php}}
